# Implementation Plan: LabLink Multi-Tenant Diagnostic Center Features

This plan outlines the architecture and tasks required to build a multi-tenant diagnostic center application where patients, doctors, and technologists collaborate within strictly isolated data boundaries.

## User Review Required

> [!IMPORTANT]
>
> Please review and answer before implementation begins:
>
> 1. **Report format**: Is there a specific format you want for the printed test reports (e.g., PDF generated by the backend, or browser-based CSS print styling)? _Assumption: browser-based CSS `@media print` backed by structured `result_data` JSON in the backend._
> 2. **Patient registration flow**: Should patients self-register before arriving, or does staff register them on arrival? _Assumption: Staff registers walk-in patients (creates a `User` + `PatientProfile`). Patients can later upgrade to full accounts with credentials._
> 3. **Notification channel**: The current `send_sms_notification` task is a mock. Should we keep it as a placeholder, or integrate a real provider (e.g., Twilio, bKash SMS)? _Assumption: Keep the mock, but wire it properly into the new `TestOrder`/`Report` lifecycle._

### Step 1.1 — Add `PatientProfile` model

**File:** `src/core/users/models.py`
**Action:** Create `PatientProfile` model linked `OneToOneField` to `User`.
**Why:** Separates patient-specific data (blood group, medical history, phone) from the generic `User` model. Lets staff create patient records without giving them login credentials initially.

```python
class PatientProfile(models.Model):
    class BloodGroup(models.TextChoices):
        A_POS = 'A+', _('A+')
        A_NEG = 'A-', _('A-')
        B_POS = 'B+', _('B+')
        B_NEG = 'B-', _('B-')
        AB_POS = 'AB+', _('AB+')
        AB_NEG = 'AB-', _('AB-')
        O_POS = 'O+', _('O+')
        O_NEG = 'O-', _('O-')

    user = models.OneToOneField(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name='patient_profile',
    )
    phone_number = models.CharField(max_length=20, blank=True)
    date_of_birth = models.DateField(null=True, blank=True)
    blood_group = models.CharField(
        max_length=3,
        choices=BloodGroup.choices,
        blank=True,
    )
    address = models.TextField(blank=True)
    medical_history = models.TextField(
        blank=True,
        help_text='Free-text medical history notes',
    )
    emergency_contact_name = models.CharField(max_length=255, blank=True)
    emergency_contact_phone = models.CharField(max_length=20, blank=True)
    registered_at_center = models.ForeignKey(
        'tenants.DiagnosticCenter',
        on_delete=models.SET_NULL,
        null=True,
        blank=True,
        related_name='registered_patients',
        help_text='The center where this patient was first registered',
    )
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        db_table = 'core_patient_profile'

    def __str__(self) -> str:
        return f'Patient: {self.user.get_full_name()}'
```

**Risk:** Low — new model, no existing data affected.

## Phase 2: Permission Classes & Multi-Tenant Enforcement

### Step 2.1 — Create permission classes

**File:** `src/core/tenants/permissions.py` _(NEW)_
**Action:** Create DRF permission classes for role-based, tenant-scoped access control.

```python
class IsCenterStaff(permissions.BasePermission):
    """User must be staff at the request's tenant center."""
    def has_permission(self, request, view):
        if not request.user.is_authenticated:
            return False
        tenant = getattr(request, 'tenant', None)
        if not tenant:
            return False
        return hasattr(request.user, 'staff_profile') and (
            request.user.staff_profile.center_id == tenant.id
        )


class IsCenterDoctor(permissions.BasePermission):
    """User must be a doctor associated with the request's tenant center."""
    def has_permission(self, request, view):
        if not request.user.is_authenticated:
            return False
        tenant = getattr(request, 'tenant', None)
        if not tenant:
            return False
        return hasattr(request.user, 'doctor_profile') and (
            request.user.doctor_profile.centers.filter(id=tenant.id).exists()
        )


class IsCenterAdmin(permissions.BasePermission):
    """User must be staff with ADMIN role at the tenant center."""
    def has_permission(self, request, view):
        if not request.user.is_authenticated:
            return False
        tenant = getattr(request, 'tenant', None)
        if not tenant:
            return False
        return (
            hasattr(request.user, 'staff_profile')
            and request.user.staff_profile.center_id == tenant.id
            and request.user.staff_profile.role == 'ADMIN'
        )


class IsCenterLabTechnician(permissions.BasePermission):
    """User must be staff with LAB_TECHNICIAN role at the tenant center."""
    def has_permission(self, request, view):
        if not request.user.is_authenticated:
            return False
        tenant = getattr(request, 'tenant', None)
        if not tenant:
            return False
        return (
            hasattr(request.user, 'staff_profile')
            and request.user.staff_profile.center_id == tenant.id
            and request.user.staff_profile.role == 'LAB_TECHNICIAN'
        )


class IsCenterStaffOrDoctor(permissions.BasePermission):
    """User must be either staff or doctor at the tenant center."""
    def has_permission(self, request, view):
        return (
            IsCenterStaff().has_permission(request, view)
            or IsCenterDoctor().has_permission(request, view)
        )
```

**Risk:** Low — new file.

### Step 2.3 — Add tenant-scoped `get_queryset()` to all ViewSets

**Files to modify:**

- `src/apps/diagnostics/views.py`
- `src/apps/appointments/views.py`

**Action:** Override `get_queryset()` to filter by `request.tenant` in every ViewSet.

**Example — `ReportViewSet`:**

```python
class ReportViewSet(viewsets.ModelViewSet):
    serializer_class = ReportSerializer
    permission_classes = [permissions.IsAuthenticated, IsCenterStaffOrDoctor]

    def get_queryset(self):
        tenant = self.request.tenant
        return Report.objects.filter(
            appointment__center=tenant,
        ).select_related(
            'test_type', 'test_order', 'appointment__patient',
        )
```

**Example — `AppointmentViewSet`:**

```python
class AppointmentViewSet(viewsets.ModelViewSet):
    serializer_class = AppointmentSerializer
    permission_classes = [permissions.IsAuthenticated]

    def get_queryset(self):
        user = self.request.user
        tenant = self.request.tenant
        qs = Appointment.objects.filter(center=tenant)

        if hasattr(user, 'doctor_profile'):
            return qs.filter(doctor=user.doctor_profile)
        if hasattr(user, 'staff_profile'):
            return qs  # Staff sees all appointments for their center
        # Patient — only their own
        return qs.filter(patient=user)
```

**Risk:** Medium — changes existing query behavior. Must ensure no cross-tenant leakage.

```python
def verify(self, request, pk=None):

    report.save(update_fields=['status', 'verified_by'])
```

```python
@receiver(post_save, sender=TestOrder)
def test_order_created_notification(sender, instance, created, **kwargs):
    if not created:
        return
    patient = instance.appointment.patient
    phone = patient.phone_number or ''
    if not phone:
        return
    message = (
        f'A {instance.test_type.name} test has been ordered for you '
        f'at {instance.center.name}.'
    )
    send_sms_notification.delay(phone, message)


@receiver(post_save, sender=Report)
def report_ready_notification(sender, instance, created, **kwargs):
    if not created:
        return
    patient = instance.appointment.patient
    phone = patient.phone_number or ''
    if not phone:
        return
    message = (
        f'Your {instance.test_type.name} report is ready. '
        f'Visit {instance.appointment.center.name} to collect it.'
    )
    send_sms_notification.delay(phone, message)
```

## Risks & Mitigations

| Risk                                        | Severity   | Mitigation                                                                                        |
| ------------------------------------------- | ---------- | ------------------------------------------------------------------------------------------------- |
| Cross-tenant data leakage                   | **High**   | Every ViewSet uses `get_queryset()` filtered by `request.tenant`. Write explicit isolation tests. |
| Report model migration breaks existing data | **Medium** | New fields are nullable. Run data migration to link existing reports if needed.                   |
| Doctor M2M center access                    | **Medium** | Permission checks always verify `doctor.centers.filter(id=tenant.id).exists()`.                   |
| Signal double-firing                        | **Low**    | Use `get_or_create()` in signals (per project rules). Guard on `created` flag.                    |
| Orphaned patients (no center)               | **Low**    | Patient queryset includes both `registered_at_center` and "has appointments at center" logic.     |

## Manual Verification Plan

1. Create 2 distinct `DiagnosticCenter`s (Center A, Center B).
2. Create Staff (Admin), Staff (Lab Tech), and Doctor for Center A.
3. Create Staff for Center B.
4. **As Staff A (Admin):** Register a patient → Create an appointment.
5. **As Doctor A:** View today's appointments → Prescribe a test (`TestOrder`).
6. **As Staff A (Lab Tech):** View pending test orders → Mark as `IN_PROGRESS` → Create a report → Upload results.
7. **As Staff A (Admin):** Verify the report.
8. **As Patient:** View report via API.
9. **As Staff B:** Verify that **none** of Center A's data (patients, doctors, test orders, reports) is visible. This is the critical multi-tenancy check.

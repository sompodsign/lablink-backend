# Database Schema Reference

**Database**: `adpp_db` | **Schema**: `adpp_db`
**Important**: Always prefix tables with `adpp_db.` (e.g., `SELECT * FROM adpp_db.core_account`)
**Last synced**: 2026-02-28
**Total tables**: 233 (excluding historical, partitioned, django/auth/celery internal)

---

## Table Naming Conventions

| Prefix | Domain | Examples |
|--------|--------|----------|
| `core_` | Core models (accounts, users, currencies) | `core_account`, `core_user`, `core_currency_code` |
| `apps_` | Feature app models | `apps_site`, `apps_app`, `apps_service_importer` |
| `ats_` | Ad Tag System | `ats_prebid_bidder`, `ats_site_setting`, `ats_version` |
| `ads_` | Ad units and orders | `ads_ad_unit`, `ads_order`, `ads_ad_size` |
| `admob_` | AdMob integration | `admob_mediation_bidder`, `admob_ad_source` |
| `pwa_` | Progressive Web App | `pwa_setting`, `pwa_push_notification` |
| `amp_` | AMP pages | `amp_setting`, `amp_web_story` |

**Historical tables**: `{table_name}_historical` (auto-generated by django-simple-history)
**Partitioned reports**: `{table_name}_y{year}m{month}` (e.g., `apps_report_y2024m01`)

## Notes

- **Encrypted fields**: `_account_info`, `_credential`, `_header_bidding_credentials` are encrypted (bytea). Cannot be read directly via SQL.
- **Historical tables**: Track changes via Django Simple History. Query `{table}_historical` for audit trail.
- **Partitioned tables**: Reports are partitioned by month. Always specify the correct partition.
- **Read-only**: Use SELECT statements only unless explicitly asked to modify data.

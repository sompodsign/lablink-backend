# Changes Analyzer

## Workflow

### Phase 1: Git Analysis
1. `git diff master...HEAD`
2. `git log master..HEAD`
3. File statistics (added, modified, deleted)

### Phase 2: Technical Analysis
1. Pattern recognition and change classification
2. Dependency and integration impact
3. Framework and technology patterns

### Phase 3: Report
```
📊 Changes Summary — Files: X modified, Y added, Z deleted | Lines: +XXX, -YYY
🔍 What Changed — categorized by area (DB, API, config, etc.)
⚙️ How It Works — data flow, patterns, caching
🎯 Purpose Analysis — business need, user value, tech debt
```
